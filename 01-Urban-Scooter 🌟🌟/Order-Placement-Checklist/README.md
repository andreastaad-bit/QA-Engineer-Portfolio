
# Order Placement Testing

This folder contains the documentation and test evidence generated during the testing of the Urban Scooter order placement flow.

## Folder Contents

### Bug Reports

The `bugs/` folder contains detailed reports of the defects identified during testing.

Each bug report includes:

* Bug summary
* Preconditions
* Steps to reproduce
* Expected result
* Actual result
* Impact
* Testing technique
* Testing environment
* Defect tracking information

The reported defects cover different areas of the order placement process, including:

* Form validation
* Date validation
* Order creation
* Order cancellation
* Order status management
* Mobile and web application interaction
* Order lifecycle and state transitions

### Test Cases

The folder also contains an Excel spreadsheet with the complete set of test cases used during the Order Placement Checklist testing process.

The spreadsheet includes the test scenarios, steps, expected results, actual results, and test execution information.

## Testing Environment

* **Web Browser:** Opera
* **Operating System:** Windows 11
* **Mobile Emulator:** Pixie 5
* **API Level:** 37
* **API Testing Tool:** Postman

The testing covered the interaction between the web application and the mobile courier application to validate the complete order placement flow.
